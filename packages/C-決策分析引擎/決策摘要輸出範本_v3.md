# 決策摘要輸出範本 v3

> 本檔提供「決策分析引擎」第四階段的最終摘要格式。
>
> 未來想調整輸出格式（例如改成不同結構、加入信心指數、加入時間戳記），直接修改本檔即可。

---

## 版本更新記錄

- v1（初版）：加權矩陣版 + SWOT 版
- v2（內部審查）：未發布
- **v3（本版本）**：新增 Pugh Matrix 版輸出範本（適用 3+ 方案的初篩結果），加入二輪精細化的引導邏輯

---

## 範本選用邏輯

| 使用的決策框架 | 對應輸出範本 |
|---|---|
| 加權決策矩陣（2 案） | **範本 A**：SCR + 加權矩陣表 |
| Pugh Matrix（3+ 案） | **範本 B**：SCR + Pugh 矩陣表（含是否需要精細化的判斷） |
| SWOT 分析（1 案） | **範本 C**：SWOT 四象限 + 綜合判斷 |
| 二輪精細化（Pugh → 加權矩陣） | **範本 A**（同 2 案情境） |

## 範本 B：Pugh Matrix 版（3 個或以上方案用）

```markdown
# 決策摘要｜[使用者的決策題目]

> 由「決策分析引擎」協助產出。這是使用 Pugh Matrix（普適矩陣）對 [N] 個方案的初步篩選結果。

---

## 情境（Situation）
[1 句話描述使用者目前所處的局面]

## 你正在比較的方案

- **A**（基準）：[名稱]
- **B**：[名稱]
- **C**：[名稱]
- （...）

---

## Pugh Matrix（普適矩陣）

> 基準是 A。其他方案在每個維度上跟基準比較，給 +（比較好）、-（比較差）、S（差不多）。

| 維度 | A（基準） | B | C | （...） |
|---|---|---|---|---|
| [維度 1] | baseline | + | - | |
| [維度 2] | baseline | - | + | |
| [維度 3] | baseline | + | + | |
| [維度 4] | baseline | S | + | |
| [維度 5] | baseline | + | - | |
| [維度 6] | baseline | + | + | |
| **淨分** | 0 | **+[N]** | **+[M]** | |

---

## 第一輪篩選結果

### 領先方案
[淨分最高的方案，並用 1 至 2 句話描述為什麼]

### 落後方案
[如果有明顯負分的方案，列出來，並描述為什麼可以先排除]

### 淨分差距判讀

| 領先方案差距 | 處理方式 | 你的狀況 |
|---|---|---|
| 差距 ≧ 3 分 | 直接採納淨分最高的方案 | ✅ / ❌ |
| 差距 0 至 2 分 | 需要二輪精細化（加權矩陣） | ✅ / ❌ |
| 所有方案都是負分 | 基準是最好的，但需要反問 | ✅ / ❌ |

---

## 建議（Resolution）

### 情況 1：領先方案明確（差距 ≧ 3 分）

**推薦：[淨分最高的方案]**

三個關鍵理由：
1. **[理由 1]**：[引用矩陣中的優勢維度]
2. **[理由 2]**：[引用維度]
3. **[理由 3]**：[引用維度]

### 情況 2：領先方案勢均力敵（差距 0 至 2 分）

**[X] 與 [Y] 兩案淨分接近，建議用加權矩陣精細比較。**

我可以直接幫你跑第二輪：你願意把 [X] 與 [Y] 當成 A 案、B 案，重新給維度權重與分數嗎？這樣可以拉出更明確的差距。

### 情況 3：所有方案都是負分

**結果顯示基準 [A] 比其他都好。**

但這個結果讓我想反問你：為什麼你會考慮其他選項？是不是 [A] 有什麼隱性問題我們沒抓到？

可能的原因：
- 你心裡其實對 [A] 有不滿，但沒表達
- 你想嘗試新的（變化的吸引力），但你列的選項都還沒成熟
- 你的維度沒有抓到真正在意的東西

要不要重抽維度？或者告訴我 [A] 的隱性問題是什麼？

---

## 風險提醒（Risk）

[依領先方案，給出 1 至 2 個最可能翻車的點 + 應對建議]

---

## 反方挑戰

讓我從兩個反方角度挑戰一下：

**反方角度 1**：
[具體論述]

**反方角度 2**：
[具體論述]

---

## 下一步建議

1. **如果領先方案明確**：執行，把方案丟給「策略規劃引擎」做 12 週計畫
2. **如果領先方案勢均力敵**：跟我說「跑第二輪」，我們用加權矩陣對前兩名做精細比較
3. **如果結果跟你直覺不符**：通常代表你心裡有未被表達的維度。告訴我哪裡怪，我重抽維度
```

## 範本 C：SWOT 版（單一方案探索情境用）

```markdown
# 決策摘要｜[決策題目]

> 由「決策分析引擎」協助產出，用 SWOT 拆解你目前正在評估的方案。

---

## 情境（Situation）
[使用者目前面對的探索性問題，1 句話]

## 你正在評估的方案
[一句話描述方案]

---

## SWOT 分析

### S Strengths（優勢，你自己的本錢）
- **[優勢 1]**：[具體描述]
- **[優勢 2]**：[具體描述]
- **[優勢 3]**：[具體描述]

### W Weaknesses（劣勢，你的弱點）
- **[劣勢 1]**：[具體描述]
- **[劣勢 2]**：[具體描述]
- **[劣勢 3]**：[具體描述]

### O Opportunities（機會，外部風口）
- **[機會 1]**：[具體描述]
- **[機會 2]**：[具體描述]
- **[機會 3]**：[具體描述]

### T Threats（威脅，外部風險）
- **[威脅 1]**：[具體描述]
- **[威脅 2]**：[具體描述]
- **[威脅 3]**：[具體描述]

---

## 綜合判斷（Resolution）

**整體傾向：[值得做 / 先暫緩 / 先補強條件再做]**

理由：
- **S × O 的綜效**：[如果做了，哪些優勢可以對接哪些機會？]
- **W × T 的風險**：[哪些劣勢遇到哪些威脅會放大成致命傷？]
- **致命因素檢查**：[W 或 T 裡有沒有「不可解」的問題？]

---

## 反方挑戰

**反方角度 1：[擇一]**
[具體論述]

**反方角度 2：[另一個]**
[具體論述]

---

## 下一步建議

依「整體傾向」給對應建議：

**若傾向「值得做」**：
1. 列出最關鍵的 W 與 T，逐一寫下「怎麼縮小或防範」
2. 把方案具體化為「另一個選項是不做」，回到加權矩陣版做精細比較
3. 把選定方案丟給「策略規劃引擎」做 12 週執行計畫

**若傾向「先暫緩」**：
1. 列出致命因素，評估是否可解
2. 給自己一個明確的「條件達標日期」（例如三個月後再重新評估）
3. 在此之前先做什麼準備能讓自己更有把握？

**若傾向「先補強條件」**：
1. 列出 3 個必須先補的 S（優勢補強）或必須先消的 W（劣勢解除）
2. 給每個補強項一個具體期限
3. 補完後回來重跑一次 SWOT
```

## 輸出時的注意事項（給 AI 的執行指引）

1. **保持簡潔**：整份摘要原則上控制在一頁（A4 列印約 800 至 1000 字）。不要為了顯得詳盡而塞滿。

2. **理由必須具體**：不要寫「整體來說 A 案比較好」這種空話。要引用矩陣裡的維度與具體分數，例如「在『個人品牌契合度』這個 5 分權重的維度上，B 案比 A 案高 2 分」。

3. **風險必須有應對**：每一個風險都要附「應對建議」，不能只列風險不給解法。

4. **反方挑戰至少兩個角度**：不可省略，這是專案包的差異化價值。

5. **Pugh Matrix 後不要硬給答案**：如果領先方案勢均力敵，要明確建議二輪精細化，不要硬給一個沒把握的結論。

6. **不要說「祝你好運」這類客套話**：直接結束在「下一步建議」即可，最後不需要結尾客套。

7. **格式優先用 Markdown 表格**：使用者複製貼上到 Notion、Obsidian、HackMD 都能直接渲染。
