# 觀點打造｜4P 框架 V1.1

## 一、4P 是什麼？

當你想寫一篇有重量的觀點作品，最大的障礙不是「不會寫」，而是「不知道自己要寫什麼」。

4P 是一套**強迫你想清楚四件事**的提問結構：

| 字母 | 名稱 | 核心問題 |
|---|---|---|
| **P1** | **Purpose 目標** | 我為什麼要說這個？我想達成什麼？ |
| **P2** | **Problem 問題** | 我要解決什麼問題？我要回答什麼疑問？ |
| **P3** | **Promise 承諾** | 讀者讀完能得到什麼具體承諾？ |
| **P4** | **Practice 實踐** | 讀者讀完可以採取什麼具體行動？ |

四個 P 都到位，你的觀點才有「可被傳遞、可被記住、可被執行」的形狀。

---

## 二、為什麼是這四個 P？

洪震宇老師在訪談裡解釋過一個現象：**很多人寫東西時，是邊寫邊想；結果寫完之後，自己也不知道剛剛寫了什麼**。

4P 的存在，就是**強迫你「寫之前先想清楚」**。

- 沒有 Purpose，你的文章沒有重心，會東一塊西一塊
- 沒有 Problem，你的文章像在自言自語，讀者不知道為什麼要看
- 沒有 Promise，你的文章看完讓人覺得「然後呢？」
- 沒有 Practice，你的文章只停在「知道」，到不了「做到」

四個 P 缺一不可，但**特別容易被跳過的是 Promise 跟 Practice**。多數人寫到 Problem 就以為寫完了，結果讀者讀完一頭霧水。

## 三、四個 P 的詳細展開

### P1 — Purpose 目標

**核心問題**：你為什麼要說這個？你想造成什麼改變？

不要回答「我想分享我的想法」這種空話。要回答：

- 「我想讓 [具體的人] 從 [原本的想法 / 行為] 變成 [新的想法 / 行為]」

範例：
- 「我想讓正在猶豫要不要轉職的工程師，從『追求安全感』變成『重新定義安全感』」
- 「我想讓剛升上主管的人，從『繼續當最強的員工』變成『學會放手讓團隊強』」

**Purpose 寫得越具體，後面三個 P 越好寫**。

### P2 — Problem 問題

**核心問題**：你要解決讀者的什麼問題？要回答他的什麼疑問？

這裡有兩種層次：

- **表面問題**：讀者自己知道、會主動搜尋的問題（例如「如何提高效率」）
- **深層問題**：讀者自己不知道、但其實是真正阻礙他的問題（例如「為什麼提高效率沒有讓他更快樂」）

**好的觀點作品，通常是回答深層問題**。因為表面問題已經有 1000 篇文章在回答了。

寫 Problem 時，問自己兩題：
1. 我的 TA 在這個議題上，他自己會說的問題是什麼？
2. 他不會說、但其實才是真正阻礙他的問題是什麼？

### P3 — Promise 承諾

**核心問題**：讀者讀完之後，能得到什麼**具體**的承諾？

注意「具體」兩個字。不要寫：
- ❌ 「讓你變得更好」
- ❌ 「讓你的生活有所改變」
- ❌ 「讓你提升思考力」

要寫：
- ✅ 「讓你下次主管退你企劃時，能問出三個讓他重新思考的問題」
- ✅ 「讓你在開會時，能用一句話打斷無效討論並把會議拉回主題」
- ✅ 「讓你 30 分鐘內，產出一份能讓主管點頭的決策備忘錄」

**測試**：你的 Promise 寫完之後，把它念給朋友聽。如果他能立刻說出「我可以用這個」，那就是好的 Promise。

### P4 — Practice 實踐

**核心問題**：讀者讀完之後，第一個可以採取的具體行動是什麼？

這呼應瓦基鐵三角的「行動三選一」：金句、工具、情境。

Practice 不一定是「動作」，也可以是：
- 一個會記住的金句
- 一個可以立刻使用的句型／模板／清單
- 一個下次遇到類似情境會想起來的判斷句

**關鍵**：Practice 必須在文章結尾**明確標出來**，不要藏在文章中段讓讀者自己找。讀者沒有那個耐心。

## 四、4P 的提問順序：為什麼是 P1 → P2 → P3 → P4？

順序非常重要，不能亂跳：

**P1（目標）→ P2（問題）**
先有目標，才能反推「為了達到這個目標，要解決什麼問題」。

**P2（問題）→ P3（承諾）**
先有問題，才能定義「我承諾解決什麼」。

**P3（承諾）→ P4（實踐）**
先有承諾，才能設計「具體該做什麼來兌現這個承諾」。

如果你倒過來寫（從 Practice 開始想），你會落入「先有結論再湊論證」的陷阱，作品會缺乏內在邏輯。

## 五、4P 怎麼跟瓦基的「鐵三角」結合？

4P 是提問結構，鐵三角是決策技術。兩者結合的方式：

| 4P | 鐵三角對應 | 結合後的提問 |
|---|---|---|
| P1 Purpose | TA | 「對誰說（狀態而非職稱）、想造成什麼改變（認知 vs 行動）、不服務誰」 |
| P2 Problem | 觀點 | 「99% 的人都這樣說、但你不同意的那一刀是什麼」 |
| P3 Promise | （獨立面向） | 「用 2-3 句話寫出主訊息」（套 Wes Kao 簡潔原則） |
| P4 Practice | 行動 | 「金句、工具、情境三選一」 |

換句話說，**4P 給你「該問哪四個面向」，鐵三角給你「每個面向該怎麼答才有力」**。

## 六、4P 檢核清單（每次寫完先過）

- [ ] **P1 Purpose**：我能用「讓 [某種人] 從 [某想法 / 行為] 變成 [新想法 / 行為]」這個句型把目標寫出來嗎？
- [ ] **P2 Problem**：我這篇回答的是「表面問題」還是「深層問題」？深層問題是什麼？
- [ ] **P3 Promise**：我的承諾夠具體嗎？讀者能立刻說「我可以用這個」嗎？
- [ ] **P4 Practice**：讀者讀完後，第一個動作是什麼？我有沒有把它明確標出來？

四題裡有任何一題答不出來，這篇文章就還沒準備好。
