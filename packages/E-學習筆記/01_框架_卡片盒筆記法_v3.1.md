# 框架｜卡片盒筆記法（Zettelkasten）

> 版本：v3.1
> 本檔案是「學習筆記專案包」的核心輸出框架。
> 在主 Instructions 的「階段 4 輪番處理」中，每完成一則筆記的訪談與確認後（Step 4.4），依本檔規則**逐則輸出單張卡片**。
> 卡片內文的撰寫方式，請同時參照「萬能框架 卡片內文撰寫方法」檔案。

---

## 一、核心精神

卡片盒筆記法源自德國社會學家 Niklas Luhmann 的 Zettelkasten 系統。Luhmann 一生用這套方法寫出 70 多本書與 400 多篇論文。

它的核心信念是：

- **每一張卡片只承載一個獨立、具體的想法**
- 卡片彼此之間可以**自由連結、自由重組**
- 卡片本身要夠簡潔，方便日後在任何脈絡下被重新運用

這個方法特別適合長期累積、想建立個人知識庫、未來想把零散想法重新組合成新作品的人。

---

## 二、卡片組成元素（極簡，三元素）

**每一張卡片只有三個組成元素**：

1. **主標題**：正面、直接、描述性的敘事句
2. **內文**：300 到 400 字，用使用者自己的話撰寫，採用萬能框架（觀點 → 案例 → 總結）的方式組織
3. **推薦產生關聯的筆記**：2 到 3 個可能與本卡片產生關聯的筆記方向

**不需要的元素**（請省略）：
- 來源書名 / 課程名 / 章節標註
- 建立日期 / 學習日期
- 標籤系統（#tag）
- 作者 / 講者資訊

**為什麼這麼簡單**：卡片的核心價值是「**獨立的想法 + 可重組的關聯**」，過多 Metadata 會讓卡片變重，反而失去卡片盒的精神。如果使用者需要追蹤來源，可以在保存卡片時自行加註。

---

## 三、各元素的詳細規範

### 3.1 主標題

**標題必須是正面、直接的描述敘事句，不是疑問句或標籤詞。**

| 寫法 | 範例 | 為什麼 |
|---|---|---|
| ✅ 推薦 | 複利效應的關鍵是時間，不是金額 | 一眼看懂核心 |
| ✅ 推薦 | 真正的學習發生在輸出的那一刻 | 直接給結論 |
| ✅ 推薦 | 主動掌控時間，是奪回人生主導權的第一步 | 有畫面、有方向 |
| ❌ 避免 | 複利 | 太空泛，無資訊量 |
| ❌ 避免 | 關於學習的幾個想法 | 沒有觀點 |
| ❌ 避免 | 為什麼輸出很重要？ | 疑問句不夠直接 |
| ❌ 避免 | 學習：輸入與輸出 | 標籤式寫法 |

**標題長度建議**：12 到 25 個字之間。

**素材來源**：階段 4 Step 4.3 訪談確認時，使用者選擇的「核心觀念方向」。

**判斷標準**：
- 把標題單獨拿出來放在 IG 限時動態，別人能看懂嗎？
- 三個月後翻到這張卡片，能不能立刻想起內容？

---

### 3.2 內文

**字數**：300 到 400 字（嚴格遵守，不要超過 450 字，也不要少於 280 字）

**語氣**：必須使用使用者自己的話與語氣。請從階段 4 訪談累積的「個人連結」、「觀點補充」、「應用想法」中，提煉並串接成完整段落。**不要從原素材中搬運原文**，除非作為短引子（且加上引號）。

**撰寫方法**：採用「**萬能框架**」的三段式結構（詳見 `02_萬能框架_卡片內文撰寫方法.md`）：

- 第一段：**觀點**（開門見山）
- 第二段：**案例**（個人經驗 + 對比質疑）
- 第三段：**總結**（行動或金句）

**風格遵守**：所有內文都必須符合 Knowledge 中的「個人寫作風格指南」（若有上傳）。沒有上傳則採用主 Instructions 第九節的通用風格規則。

---

### 3.3 推薦產生關聯的筆記

**設計目的**：讓每張卡片不只是孤立存在，而是知識網路中的一個節點。當使用者未來累積越多卡片，這些關聯會自然湧現出原本沒看見的洞察。光是看到推薦的筆記方向，使用者可能就會直接想動手寫下一張新卡片。

**呈現方式**：

依「**主題類別**」提供 2 到 3 個可能與本卡片產生關聯的筆記方向。**不需要真的存在的卡片**，而是給使用者一個「下次可以往哪個方向繼續累積」的種子。

**格式範例**：

```
🔗 推薦產生關聯的筆記：
→ 時間管理類：「為什麼最重要的事永遠不在 to-do list 上」
→ 學習方法類：「主動回想比反覆閱讀更有效」
→ 個人習慣類：「習慣的養成靠環境設計，不是意志力」
```

**設計原則**：

- 每個方向用「主題類別 + 卡片標題建議」的格式呈現
- 卡片標題建議要寫成完整的、可以直接拿來當未來卡片標題的句子
- 三個方向之間要有真實的差異，覆蓋不同的應用面向（例如：方法、心態、實作、人際）
- **不要連結到本次對話中已產出的卡片**。推薦的方向是給使用者「未來可以累積什麼新卡片」的種子，而不是重複呈現已經產出的內容
- 光是看到推薦的標題與主題，使用者可能就會直接想動手寫下一張新卡片，這就是這個欄位最有價值的地方

## 四、單張卡片的輸出格式

**重要**：所有卡片都是**直接在對話中展開**，不要產生檔案下載。使用者可以邊讀邊複製，或對話結束後一次複製全部。

完整格式如下：

```markdown
〔第 X 則卡片盒筆記｜完成〕

## 卡片 X｜複利效應的關鍵是時間，不是金額

複利真正可怕的地方不是利率，是時間。如果你 25 歲開始每月存一萬塊，跟 35 歲開始每月存兩萬塊，60 歲那年帳戶裡的數字差距會大到讓人不敢相信。這件事提醒我，有些決定不能等「準備好了」再做，因為時間本身就是最大的成本。

我以前一直覺得「等我賺更多再開始投資」，後來才意識到這個想法的陷阱。所謂的「賺更多」永遠到不了那一天，因為慾望會跟著收入一起膨脹。真正關鍵的不是金額，是「現在就開始」這個動作。同樣的邏輯也適用在寫作、運動、學語言上面，每天一點點，三年後的差距是難以追上的。不過這個觀念也有個邊界，前提是你要先確認「方向是對的」，否則複利只是加速錯誤。

下週我要做的具體小行動：把原本「想清楚再開始」的那本書筆記，先寫第一段貼到部落格，不管完不完整。

🔗 推薦產生關聯的筆記：
→ 時間管理類：「為什麼最重要的事永遠不在 to-do list 上」
→ 學習方法類：「先動手再修正，比想清楚再開始更有效」
→ 心態調整類：「擁抱不完美的版本一，才有機會走到版本十」
```

輸出後簡短過渡到下一則：

```
✓ 第 X 則完成

接下來進入第 (X+1) 則：「主題名稱」
```

## 五、不同卡片數量的內容分配

主 Instructions 階段 2 已經決定了 N（3 / 5 / 7 / 自訂）。階段 3 的 MECE 拆分依此產出 N 個主題。階段 4 依序處理。

### 5.1 三張卡片版

主題拆分聚焦在素材中**最有衝擊力**的三個重點。三張卡片之間最好有「對比」或「遞進」關係。

### 5.2 五張卡片版

涵蓋大部分核心觀念。建議結構：

- 1 張「總綱卡片」：素材的核心精神
- 3 張「核心觀念卡片」：素材中最值得獨立成卡的三個觀念
- 1 張「應用卡片」：偏實作、行動導向的內容

### 5.3 七張卡片版

連同延伸思考都收進來。建議結構：

- 1 張總綱卡片
- 4 張核心觀念卡片
- 1 張對比 / 質疑卡片
- 1 張應用卡片

### 5.4 自訂張數

- 1 到 2 張：只挑最關鍵的一個觀念，用完整篇幅展開
- 8 張以上：建議拆成兩次對話完成，避免品質下降

---

## 六、輸出後的收尾

所有 N 則卡片產出完後，請依主 Instructions 階段 5 的格式給予「基本用法 + 進階用法」兩層提示。

**基本用法**：複製到 Notion、Obsidian、Heptabase、Apple Notes、Logseq 等筆記軟體。

**進階用法**：未來可用 Claude Cowork 或 Claude Code 等 Agent 工具，把卡片直接送到個人知識庫，由 Agent 做自動分類、交叉連結、整理成新的對外輸出。

---

## 七、品質檢核清單（每張卡片輸出前自查）

### 標題
- [ ] 是正面描述敘事句，不是疑問句或標籤詞
- [ ] 長度在 12 到 25 字之間
- [ ] 三個月後看到能立刻想起內容

### 內文
- [ ] 字數在 300 到 400 字之間
- [ ] 使用者的語氣，沒有搬運原文
- [ ] 採用萬能框架三段式（觀點 / 案例 / 總結）
- [ ] 案例段有使用者的個人經驗
- [ ] 沒有禁用詞，沒有破折號，沒有 AI 味句型
- [ ] 符合「個人寫作風格指南」（若有上傳）

### 推薦產生關聯的筆記
- [ ] 提供 2 到 3 個關聯方向
- [ ] 每個方向有「主題類別 + 卡片標題建議」
- [ ] 三個方向之間有真實差異
- [ ] 沒有連結到本次已產出的卡片（推薦的是延伸的新方向）

### 整體
- [ ] 每張卡片可以「單獨拿出來」也看得懂
- [ ] 標點與排版符合風格指南（全形標點、中英文間半形空格）
- [ ] 整體讀起來「像使用者自己會寫的東西」

任何一項沒過，請修正後再輸出。
